Terminology

We can distinguish variables and objects.

  • Variable generally refers to instances of scalar data types.
  • Objects generally refers to instances of other types.

We can also distinguish two kinds of objects: value types and reference types:

  • Value types refer to types that can be copied.
  • Reference types refer to types that cannot be copied.

Value types can be POD (Plain Old Data). A POD type does not have any non-POD members, constructors, destructors, and virtual table.

Primitive Types

A signed integral type and its unsigned counterpart are always the same size, but they differ in how the bit pattern is interpreted.

Floating point

  • Trade-off between range and precision.
  • Represented approximately to a fixed number of significant digits and scaled using an exponent in some fixed base.
  • The term floating point refers to the fact that a number's radix point (or decimal point) can be placed anywhere relative to the significant digits of the number.

IEEE 754

In most languages, floating point types are represented by the IEEE 754 rules.

  • Divide by 0 produces +/- INF, except 0/0 which results in NaN.
  • INF - INF = NaN
  • (+/-)INF / (+/-)INF = NaN
  • (+/-)INF * 0 = NaN
  • Operation when either or both operands is NaN returns NaN
  • Comparisons (except Not Equal) when either or both operands is NaN returns FALSE
  • Not Equal comparison when either or both operands is NaN returns TRUE
  • +0 equals -0
